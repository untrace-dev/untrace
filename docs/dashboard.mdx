## Getting Started

### Accessing the Dashboard

1. Navigate to [https://acme.sh/app](https://acme.sh/app)
2. Sign in with your Acme account
3. You'll land on the main dashboard showing your webhook activity

### Dashboard Layout

The dashboard is organized into several key sections:

- **Navigation Sidebar**: Quick access to all dashboard features
- **Main Content Area**: Displays your selected view (webhooks, history, settings, etc.)
- **Activity Feed**: Real-time webhook activity stream
- **Status Bar**: Connection status and notification indicators

## Real-time Monitoring

### Live Webhook Feed

The live feed shows webhooks as they arrive at your endpoints:

<Tabs>
  <Tab title="List View">
    View webhooks in a chronological list with key information:
    - Timestamp
    - Provider (Stripe, GitHub, etc.)
    - Event type
    - Status code
    - Response time
    - Target destination
  </Tab>
  <Tab title="Detail View">
    Click any webhook to see detailed information:
    - Full request headers
    - Request body (with JSON formatting)
    - Response data
    - Timing breakdown
    - Error details (if any)
  </Tab>
</Tabs>

### Filtering and Search

Filter the live feed to focus on what matters:

```javascript
// Example filters
{
  provider: "stripe",
  status: "success",
  destination: "dev-1",
  timeRange: "last-hour"
}
```

Available filter options:
- **Provider**: Filter by webhook source (Stripe, GitHub, Clerk, etc.)
- **Status**: Success, Failed, Pending
- **Destination**: Filter by team member or endpoint
- **Time Range**: Last hour, 24 hours, 7 days, custom range
- **Event Type**: Provider-specific event types

## Request History

### Searching Past Webhooks

Access historical webhook data with powerful search capabilities:

1. Navigate to the **History** tab
2. Use the search bar for text-based searches
3. Apply filters for precise results
4. Export results as CSV or JSON

### Request Details

Each historical request includes:

<AccordionGroup>
  <Accordion title="Request Information">
    - Unique request ID
    - Timestamp (with timezone)
    - Provider and event type
    - HTTP method and path
    - Source IP address
  </Accordion>
  <Accordion title="Headers">
    - All HTTP headers
    - Provider signatures
    - Content type and encoding
    - Custom headers
  </Accordion>
  <Accordion title="Payload">
    - Full request body
    - JSON/XML formatting
    - Binary data indicators
    - Payload size
  </Accordion>
  <Accordion title="Response">
    - Status code
    - Response headers
    - Response body
    - Processing time
    - Error messages
  </Accordion>
</AccordionGroup>

### Replay Requests

Replay any webhook request for debugging:

1. Find the request in history
2. Click the **Replay** button
3. Optionally modify the payload
4. Select the destination
5. Send the replay

<Note>
Replayed requests are marked with a special indicator to distinguish them from original webhooks.
</Note>

  <Step title="Configure Security">

## Analytics and Insights

View key metrics at a glance:

- **Total Requests**: Daily, weekly, monthly counts

- **Response Time**: Average, P95, P99 latencies

## Advanced Features

### Webhook Transformations

Transform webhook payloads before delivery:

```javascript
// Example transformation
{
  "transform": {
    "remove": ["sensitive_field"],
    "rename": {
      "old_field": "new_field"
    },
    "add": {
      "timestamp": "{{now}}",
      "environment": "production"
    }
  }
}
```

### Conditional Routing

Route webhooks based on conditions:

```yaml
rules:
  - condition: "payload.amount > 1000"
    destination: "high-value-processor"
  - condition: "headers['X-GitHub-Event'] == 'push'"
    destination: "ci-pipeline"
  - condition: "default"
    destination: "standard-processor"
```

### Bulk Operations

Perform operations on multiple webhooks:

- Bulk replay
- Bulk export
- Bulk delete
- Bulk status updates

## API Access

Access dashboard functionality programmatically:

```bash
# Get webhook history
curl -X GET https://api.acme.sh/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY"

# Replay a webhook
curl -X POST https://api.acme.sh/webhooks/{id}/replay \
  -H "Authorization: Bearer YOUR_API_KEY"

# Get team statistics
curl -X GET https://api.acme.sh/stats \
  -H "Authorization: Bearer YOUR_API_KEY"
```

See the [API Reference](/api-reference/introduction) for complete documentation.
